# Add questions to dataset

Integrate new questions generated by Gemini into the BJJ-VQA dataset.

## Context

Questions are generated externally by Gemini from CC-licensed instructional videos. Gemini outputs structured question blocks with timestamps. The human contributor extracts frames manually using a screenshot tool and provides the images. Your job is to assign IDs, update samples.json, and validate.

Schema is the source of truth: see `src/bjj_vqa/schema.py`. Allowed values for `experience_level`, `category`, and `subject` are defined there as Literal types. Do not invent values.

Steps can be parallelized where dependencies don't exist (e.g., run Step 1 and Step 2 together).

## Step 1 — determine the next ID

Read `data/samples.json` and find the highest existing numeric ID. All new IDs increment from there, zero-padded to 5 digits.

```bash
uv run python -c "
import json
data = json.load(open('data/samples.json'))
print(max(int(r['id']) for r in data))
"
```

## Step 2 — verify frame files

The contributor has already extracted frames at the timestamps from the Gemini output using their screenshot tool. Confirm the image files are present in `data/images/` with the correct 5-digit names before proceeding.

```bash
ls data/images/ | tail -10
```

If images are missing, ask the contributor to add them before continuing.

## Step 3 — build the JSON records

From each Gemini question block, construct a record matching this shape exactly:

```json
{
  "id": "00002",
  "image": "images/00002.jpg",
  "question": "<question stem only, no answer letter>",
  "choices": ["<A>", "<B>", "<C>", "<D>"],
  "answer": "<A|B|C|D>",
  "experience_level": "<beginner|intermediate|advanced>",
  "category": "<gi|no_gi>",
  "subject": "<guard|passing|submissions|controls|escapes|takedowns>",
  "source": "https://www.youtube.com/watch?v=<VIDEO_ID>&t=<SECONDS>s"
}
```

Strip these from Gemini output before appending: option type labels in brackets like `[CORRECT]`, `[WRONG-CONTEXT]`, etc., and any review-only fields. The `choices` array must have exactly 4 strings in A/B/C/D order.

## Step 4 — verify frames

Read the images at the paths you just assigned to verify they show the correct position. The frames should clearly display the technique detail referenced in the question. If a frame is wrong, ask the contributor to re-extract at the correct timestamp.

## Step 5 — append and validate

Append all new records to `data/samples.json` (keep existing records intact), then run:

```bash
uv run bjj-vqa validate
```

Fix any errors before committing. Common issues: wrong field value not in the Literal enum, image file missing, choices array not exactly 4 items.

## Step 6 — update attributions

If this is a new video source (not already in `ATTRIBUTIONS.md` and `sources/registry.jsonl`), add entries to both.

**ATTRIBUTIONS.md** — add a row to the Sources table:

| Title | Author | Source | License | Used in |
|-------|--------|--------|---------|---------|
| [Video Title](https://youtube.com/watch?v=VIDEO_ID) | Channel Name | [YouTube](https://youtube.com/watch?v=VIDEO_ID) | [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/) | FIRST_ID-LAST_ID |

Use the TASL framework: Title, Author, Source, License. All four should have links. The "Used in" column shows which question IDs use this video.

**sources/registry.jsonl** — add a JSON line:

```json
{"url": "https://youtube.com/watch?v=VIDEO_ID", "title": "...", "creator": "...", "license_type": "cc_by", "question_ids": ["XXXXX", ...]}
```

Then run:

```bash
uv run bjj-vqa validate-sources
```
