# Domain Docs

Before exploring this repo, read:

- `CONTEXT.md` at the repo root — domain model, architecture, question structure, quality evals, CLI
- `docs/adr/` — read ADRs relevant to the area you're working in

If these files don't exist, proceed silently.

When naming concepts in issues, tests, or code, use the vocabulary from `CONTEXT.md`. If you contradict an existing ADR, surface it explicitly rather than silently overriding it.
